New/Edit Transaction Modal

Date: Memo: Category: Amount: Split: Recurring:
  • Daily
  • Weekly
  • Monthly

Endpoints:

  • /transaction/{accountHolderId}/{transactionId}
    • Description: returns transaction details to be either viewed or edited. There are kinds of transactions. a single transaction, or a recurring transaction template. A single transaction only has one category and ammount, and does not have a recurringObject. A template will have an array of one or more category/ammount pairs, representing a split transaction (think a transfer from one category to another, or a recurring deposit where 90% goes into one category and 10% goes into tithing category)
    • Method: Get
    • Parameters Passed: (accountHolderId, transactionId)
    • Expected Return:
      • Success: JSON object with {transactionId, date, memo, [category,ammount], recurringObject}
      • Failure: {error message}
  • /transaction/edit/{accountHolderId}/{transactionId}
    • Description: for editing single transactions or transaction templates
    • Method: Post
    • Parameters Passed: (accountHolderId, transactionId, newTransactionObject)
    • Expected Return:
      • Success: {success message}
      • Failure: {error message}
  • /transaction/new/{accountHolderId}/{transactionId}
    • Description: adding new single transactions or transaction templates
    • Method: Post
    • Parameters Passed: (accountHolderId, transactionId, newTransactionObject)
    • Expected Return:
      • Success: {success message}
      • Failure: {error message}
