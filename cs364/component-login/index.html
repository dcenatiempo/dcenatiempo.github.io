Endpoints:

  • /login
    • Description: After client validation, credentials are posted to server to be authenticated
    • Method: Post
    • Parameters Passed: (userName, password)
    • Expected Return:
      • Success: Serve up Banker or Account Holder page, cookie with authentication token & userId
      • Failure: {error message}
  • /login/register
    • Description: After client validation, credentials are posted to server to be authenticated. Server will either create new user, or return reason why it couldn't
    • Method: Post
    • Parameters Passed: (userName, password)
    • Expected Return:
      • Success: Serve up Banker or Account Holder page, cookie with authentication token & userId
      • Failure: {error message}
  • /login/oauth
    • Description: Login/Register with twitter/facebook/google
    • Method: Post
    • Parameters Passed: (?)
    • Expected Return:
      • Success: Serve up Banker or Account Holder page, cookie with authentication token & userId
      • Failure: {error message}
